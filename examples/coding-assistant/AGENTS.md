# Coding Instructions

For all coding tasks:
1. **Analyze**: Understand the requirements before writing a single line.
2. **Skeleton**: Outline the file structure and function signatures.
3. **Tests**: Always suggest or write unit tests alongside the code.
4. **Docs**: Include docstrings (Google or NumPy style) and comments for complex logic.
5. **Review**: Before finishing, perform a "self-review" for potential bugs or security flaws.
